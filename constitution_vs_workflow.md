# 憲法（Rules）とワークフロー（Workflows）の違い

## 1. 定義と関係性

> **憲法（Rules / GEMINI.md）**
>
> * **定義**: AIエージェントの「行動原理」や「禁止事項」を定めた最高法規。
> * **役割**: 「何をすべきか」「何をすべきでないか」という**判断基準**を提供する。
> * **変更頻度**: 低い（基本思想が変わらない限り不変）。
> * **例**: 「日本語で話せ」「破壊的変更をするな」「専門用語には定義を添えろ」

> **ワークフロー（Workflows）**
>
> * **定義**: 特定のタスクを遂行するための「具体的な手順書」や「マニュアル」。
> * **役割**: 「どのように行うか」という**操作手順**を提供する。
> * **変更頻度**: 高い（ツールや環境の変化に合わせて更新される）。
> * **例**: 「デプロイ手順」「テスト実行方法」「新機能追加フロー」

---

## 2. わかりやすい対比

| 項目 | 憲法 (Rules) | ワークフロー (Workflows) |
| :--- | :--- | :--- |
| **イメージ** | 国の憲法、社訓、行動規範 | 業務マニュアル、料理のレシピ |
| **問い** | Why (なぜ), What (なに) | How (どのように) |
| **違反時** | 信頼を損なう (重大な過失) | 作業が失敗する (やり直し可能) |
| **抽象度** | 高い (思想寄り) | 低い (作業寄り) |
